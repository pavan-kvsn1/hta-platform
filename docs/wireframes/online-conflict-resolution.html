Implementation flow summary:
  • Step 1: User clicks Save → API returns 409 with { serverVersion, serverTimestamp, updatedBy }
  • Step 2: Frontend fetches the latest certificate data from the server
  • Step 3: Deep-diff the user's form data against the server version to find changed fields
  • Step 4: If ≤3 conflicts → show compact dialog (Page 3). If ≥4 → show full merge (Page 2).
  • Step 5: User picks per field. On "Save Merged Version", POST the merged payload with the server's updatedAt as the new base version.
  • Step 6: If another conflict happens during merge-save (unlikely), repeat the flow.
